## 用語<br>
- ステートメント<br>
SQLで言うなら「SELECT」や「INSERT」「DELETE」などで一つのクエリを構築しますが、この構築された文の全体をステートメントと言います。<br>
ステートメントの種類（データ定義言語 / データ操作言語）<br>

- データ定義言語（DDL）<br>
データベース構造の作成や変更、削除といった機能を担っています。<br>
ALTER<br>
照合順序<br>
CREATE<br>
DROP<br>
DISABLE TRIGGER<br>
ENABLE TRIGGER<br>
RENAME<br>
UPDATE STATISTICS<br>

- データ操作言語（DML）<br>
データベースに格納されるデータを挿入、更新、変更、削除するためのもの<br>
BULK INSERT<br>
DELETE<br>
INSERT<br>
SELECT<br>
UPDATE<br>
MERGE<br>
TRUNCATE TABLE<br>

- sqlインジェクション<br>
Webアプリケーションに対し不正なSQL文を注入して不正に操作する攻撃手法<br>

- 物理削除<br>
SQLでDELETE文を発行してレコードを削除すること<br>
原則として容易にデータの復活はできません（バックアップから復元することはできます）<br>

- 論理削除<br>
テーブルに削除フラグを用意し、TRUE/FALSEで削除をされたか管理する方法<br>

- スケールアップ<br>
サーバーのCPUやメモリーなどハードウエアの増設によって処理能力を向上させること<br>

- スケールアウト<br>
システムを構成するサーバーの台数を増やして処理能力を向上させること<br>
何らかのトラブルでサーバーが故障しても別のサーバーでカバーできる<br>
機器の数だけ構築作業や設計変更の手間が増え、保守する費用も台数分かかる<br>

- ロードバランサー<br>
サーバーにかかる負荷分散を行うデバイス（機器）<br>

- トラフィック量<br>
ネットワーク回線で送受信される通信データ量<br>

- ハードウェア<br>
物理サーバやネットワーク機器（モニター、ハードディスク、プリンタ、 スキャナ、キーボード、マウス）<br>

- ソフトウェア<br>
コンピュータやスマートフォン、その他電子機器に搭載されたプログラム<br>

- カーネル<br>
オペレーティングシステム（OS）の基本機能の役割を担うソフトウェア<br>

- RDBMS（リレーショナルデータベース管理システム）<br>

## 開発工程<br>
要件定義→外部設計→内部設計→開発→単体テスト→結合テスト→システムテスト→運用テスト→システム移行<br>

■要件定義<br>
開発プロジェクトを進める前に、顧客にとって必要な機能や要件をまとめる作業<br>
すべての要求を実現できるとは限らないため、ユーザーと話し合い要件に優先順位をつける<br>

■外部設計（基本設計）<br>
要件定義で作成した内容をもとに、外見的な見た目の部分を設計する<br>
<---------方式設計---------><br>
・システムに使用するソフトウェア、アプリケーション<br>
・開発に使用する言語やフレームワーク<br>
<---------機能設計---------><br>
・画面遷移設計<br>
開発しようとしているソフトウェアの全体の画面遷移図を記載する<br>
・画面設計<br>
画面を構成するレイアウト、各コンポーネントの表示仕様、入力仕様、操作仕様をなどを記載する<br>
※実際の画面をイメージできて、操作方法が分かるように記載する<br>
※画面のデザインを行う必要はない<br>
※ワイヤーフレームとも呼ばれることがある<br>
<--------その他設計--------><br>

■内部設計（詳細設計）<br>

■単体テスト（UT:UnitTest）<br>
クラスや関数単位でプログラムが正常に動くかを検証するもの<br>
<---------ホワイトボックステスト---------><br>
・開発者視点のテスト<br>
・内部のプログラムが意図したとおりに動作しているかを確認する<br>
<----制御フローテスト----><br>
・分岐条件に対して処理が正しく動作するかを確認するテスト<br>
１：命令網羅（ステートメントカバレッジ C0カバレッジ）<br>
２：分岐網羅（ブランチカバレッジ C1カバレッジ）<br>
３：条件分岐（デシジョンカバレッジ C2カバレッジ）<br>
<----データフローテスト----><br>
・<br>
<---------ブラックボックステスト---------><br>
・ユーザー視点のテスト<br>
・プログラムの内部構造を考慮せずに、外部から網羅的に確認する<br>
・画面表示の不具合やレイアウト崩れなど、正しい出力ができているか確認する<br>
<---------グレーボックステスト---------><br>
・ホワイトボックステストとブラックボックステストの中間<br>
・内部構造を理解した上で外部からの機能や仕様を確認する<br>
・内部の構造についてよく理解しているプログラム作成者がテスト実施する<br>

■結合テスト（IT:IntegrationTest）<br>

■総合テスト（ST:System Test）<br>

■運用テスト<br>
■シナリオテスト<br>
■アドホックテスト（モンキーテスト）<br>
